# Backlog

Ideas and features for future consideration. Items here are not committed to any release - they're captured so they aren't forgotten.

## How to Use

- **Add items**: Run `/backlog add <title>` to add new ideas
- **Promote items**: When ready to build, create an epic with `/feature` or `/epic`
- **Mark done**: Change `[ ]` to `[x]` when promoted or decided against
- **Review regularly**: During planning, scan for items that should move to active work

## Legend

**Categories:**
- `feature` - New user-facing functionality
- `enhancement` - Improvement to existing feature
- `tech-debt` - Code/architecture improvements
- `research` - Needs investigation before committing
- `ux` - User experience improvements
- `perf` - Performance optimization

**Priority:**
- `high` - Would significantly improve the product
- `medium` - Valuable but not urgent
- `low` - Nice to have someday
